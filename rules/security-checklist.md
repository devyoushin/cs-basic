# 보안 검토 체크리스트

CS 문서에서 보안 관련 내용 작성 시 확인해야 할 항목입니다.

---

## 1. 코드 예시 보안

- [ ] 하드코딩된 패스워드/API 키 금지
- [ ] SQL 인젝션 취약한 코드 단독 제시 금지 (보안 예시로 제시 시 명확히 표시)
- [ ] 안전하지 않은 랜덤 함수 보안 맥락 사용 금지

## 2. 보안 개념 정확성

- [ ] 암호화 알고리즘 권장: AES-256, RSA-2048+, SHA-256+
- [ ] 사용 금지 알고리즘 명시: MD5, SHA-1 (보안 목적)
- [ ] TLS 1.2+ 사용 권장 (TLS 1.0/1.1 deprecated 언급)

## 3. 네트워크 보안 예시

- [ ] `0.0.0.0/0` 허용 설정에 주의 경고
- [ ] 평문 프로토콜(HTTP, FTP) 사용 지양 명시

## 4. 금지 표현

- 실제 서버 IP, 도메인, 계정 정보 노출 금지
- 실제 개인정보 예시 데이터 사용 금지
- 모든 민감 정보는 `<YOUR_VALUE>` 형식으로 대체
