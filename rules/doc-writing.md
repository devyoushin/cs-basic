# 문서 작성 규칙 (Doc Writing Rules)

Claude가 이 저장소에서 문서를 작성하거나 수정할 때 반드시 따라야 할 규칙입니다.

---

## 1. 언어 및 문체

- **기본 언어**: 한국어
- **영어 원문 병기**: 기술 용어는 첫 등장 시 반드시 원문 병기
  - 예: 가상 메모리 (Virtual Memory), 가비지 컬렉션 (Garbage Collection)
- **문체**: 간결한 서술체 (`~다.`, `~한다.`)
- **경어체 금지**: `~합니다` 사용하지 않음

## 2. 코드 블록

- **언어 태그 필수**:
  - ` ```bash ` — 터미널 명령어
  - ` ```java ` — Java 코드
  - ` ```python ` — Python 코드
  - ` ```sql ` — SQL 쿼리
  - ` ```c ` — C/C++ 코드
- **실행 가능한 코드**: 복붙해서 바로 실행 가능한 수준
- **주석**: 핵심 로직에 한국어 주석 추가

## 3. 섹션 구성 필수 요소

| 섹션 | 내용 | 생략 가능 여부 |
|------|------|----------------|
| 1. 개요 | 정의 + 중요성 | 불가 |
| 2. 핵심 설명 | 원리, 예시, 면접 포인트 | 불가 |
| 3. 실무 포인트 | 실제 적용 방법 | 불가 |
| 4. 심화 내용 | Senior 레벨 심화 | 가능 |
| 5. TIP | 팁 및 주의사항 | 가능 |

## 4. 면접 포인트 형식

| 질문 | 핵심 답변 |
|------|---------|
| Q: {예상 면접 질문} | A: {한 문장 핵심 답변} |

## 5. 금지 사항

- 추측성 내용 작성 금지 (RFC, 공식 문서, 검증된 사례 기반만)
- 오래된 정보 작성 금지 (Java 8 이상, Linux 5.x 이상 기준)
- 불확실한 수치 단독 기재 금지
