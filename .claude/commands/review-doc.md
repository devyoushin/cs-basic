아래 기준으로 `$ARGUMENTS` 문서를 검토하고 개선점을 제안해 주세요.

## 검토 항목

### 1. 구조 완성도
- [ ] 필수 섹션(개요/핵심설명/실무포인트) 모두 존재하는지
- [ ] 면접 포인트 섹션이 있는지

### 2. 기술적 정확성
- [ ] 내용이 최신 기술 스택을 반영하는지
- [ ] 공식 문서/RFC/논문 기반 내용인지
- [ ] 잘못된 기술 정보가 없는지

### 3. 실무 연계성
- [ ] 면접에서 실제로 나오는 질문을 다루는지
- [ ] 운영 환경에서 적용 가능한 내용인지

## 출력 형식

```
## 검토 결과: {파일명}

### 잘된 점
- ...

### 개선 필요 사항
1. [섹션명] 문제 → 개선 방안

### 기술적 오류 (있을 경우)
- 내용: ...
- 수정 방향: ...
```
